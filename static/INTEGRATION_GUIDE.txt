## 2. Configuration Options

Customize the widget behavior and appearance with these options:

### Required Options

| Option | Type | Description |
|--------|------|-------------|
| `merchantEmail` | string | Your email address for support notifications |
| `shopName` | string | Your business/shop name displayed in the chat |
| `shopDomain` | string | Your website domain |

### Optional Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `apiBaseUrl` | string | `https://primecaves-chatbot.floot.app` | Backend API URL (use default) |
| `primaryColor` | string | `#4F8EF7` | Main theme color for the widget |
| `agentName` | string | `'Support Admin'` | Name displayed for support agent |
| `widgetPosition` | string | `'bottom-right'` | Position: `'bottom-right'` or `'bottom-left'` |
| `autoOpen` | boolean | `false` | Automatically open widget on page load |
| `initialMessage` | string | `undefined` | Auto-send this message when chat starts |
| `hideContactForm` | boolean | `false` | Skip contact form (requires all user details) |

## 3. API Methods

Once initialized, you can control the widget programmatically using these methods:

### `open()`
Opens the chat widget dialog.
### `close()`
Closes the chat widget dialog.
### `openWithMessage(message)`
Opens the widget and auto-sends a message.
### `destroy()`
Completely removes the widget from the page.

## 6. Platform-Specific Notes

### Shopify
- Add the script to `theme.liquid` before `</body>`
- Use Liquid tags for dynamic values: `{{ shop.email }}`, `{{ shop.name }}`
- Works with all Shopify themes (Dawn, Debut, etc.)

### WordPress
- Add via theme files (`footer.php`) or plugins
- Use PHP functions: `get_option('admin_email')`, `get_bloginfo('name')`
- Compatible with all page builders (Elementor, Gutenberg, etc.)

### React/Next.js/Vue
- Load script dynamically in `useEffect` or `mounted` lifecycle
- Remember to call `destroy()` on component unmount
- TypeScript: Add `declare global { interface Window { PrimeCavesChatWidget: any } }`

### Plain HTML
- Just paste the script tags before `</body>`
- No build tools or npm required
- Works with any HTML file

## 7. Troubleshooting

### Widget not appearing?
- Check browser console for errors
- Verify script URL is correct: `https://primecaves-chatbot.floot.app/_api/widget-sdk`
- Ensure `init()` is called after script loads
- Check that `merchantEmail`, `shopName`, and `shopDomain` are provided

### Widget blocked by ad blockers?
- The widget should not be blocked, but some aggressive blockers may interfere
- Ask users to whitelist your domain

### Need to reinitialize?
- Call `destroy()` first, then `init()` again with new config

### CORS issues?
- The widget is designed to work cross-origin
- Ensure you're using the production URL: `https://primecaves-chatbot.floot.app`
